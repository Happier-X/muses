# 专项优化：SongsPage 多选与批量操作控制层拆分

## 1. 背景

在完成 `SongsPage` 可见列表控制器和封面预取协调器拆分之后，页面中仍然保留了一大块与多选、批量操作相关的状态与交互逻辑。

主要包括：

- 多选开关状态
- 已选歌曲集合
- 全选/取消全选
- 点击歌曲时的多选分支处理
- 收藏到歌单入口
- 批量移除入口

这些内容虽然和页面交互强相关，但本质上已经属于独立的控制层职责，不适合长期继续堆在页面主体中。

## 2. 本次目标

本次拆分的目标是：

- 提取多选状态和基础操作逻辑
- 页面只负责把当前列表和 UI 事件交给 controller
- 为后续继续拆“批量删除执行器”和“批量加入歌单动作”打基础

## 3. 本次拆分范围

本次纳入 controller 的内容：

- 多选状态开关
- 已选 ID 集合
- 单个条目的选择切换
- 全选/取消全选
- 选中数量计算

暂不完全下沉的内容：

- 实际的删除执行流程
- 批量加入歌单后的业务反馈
- 播放队列插入动作

## 4. 预期收益

- `SongsPage` 页面逻辑继续减重
- 多选状态变化有统一入口
- 后续继续拆批量动作时，能直接围绕 controller 扩展

## 5. 验收标准

- 新增 `SongsSelectionController`
- 页面不再直接维护主要的多选切换逻辑
- 多选行为保持一致
- `flutter analyze` 通过
- 基础测试通过
