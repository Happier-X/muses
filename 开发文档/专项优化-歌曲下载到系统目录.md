# 专项优化 - 歌曲下载到系统目录

## 背景

当前应用已经具备远程歌曲缓存能力，但缓存文件位于应用私有目录，主要服务播放提速，用户无法直接在系统文件管理器中把它当作“已下载歌曲”使用。

用户希望在此基础上补齐真正的下载能力：

1. 支持一键保存到系统公共目录，例如 `Download/NagoMusic`
2. 支持用户在设置中配置自定义目录保存
3. 尽量复用现有缓存下载链路，避免重复下载

## 目标

本次下载功能目标分两层：

### 1. 系统下载目录

- 为歌曲提供“下载到本地”入口
- Android 下默认保存到 `Download/NagoMusic`
- 保存后能在系统文件管理器中直接看到

### 2. 自定义目录

- 在设置页提供统一下载路径配置
- 用户可通过系统目录选择器选择目标文件夹
- 歌曲面板只保留一个“保存到本地”入口，实际保存位置由设置决定

## 设计原则

### 1. 缓存与下载分离

- 缓存仍用于播放加速
- 下载表示显式导出到用户可见目录
- 两者不混为同一个概念

### 2. 复用已有下载链路

- 远程歌曲优先检查现有完整缓存
- 已缓存则直接复制到目标目录
- 未缓存再走现有音频下载逻辑

### 3. 本地歌曲特殊处理

- 本地歌曲本身已经存在于系统文件中
- 第一版不做“重复复制到下载目录”的强制行为
- 如果用户主动选择导出，则允许复制到目标目录

## 实施方案

### A. 新增下载服务

- 新建独立 `SongDownloadService`
- 负责：
  - 解析歌曲文件名
  - 复用缓存或触发下载
  - 复制到系统下载目录
  - 复制到用户自定义目录
  - 处理重名、非法字符与错误提示

### B. Android 公共下载目录支持

- 第一版优先使用 Android 公共 `Download/NagoMusic`
- 路径通过平台通道获取，避免 Dart 层硬编码平台差异
- 后续若需要更严格适配 Android 10+ scoped storage，可再继续升级为 MediaStore 写入方案

### C. 自定义目录导出

- 在设置页选择并保存下载目录
- 下载动作执行时读取当前设置路径
- 文件名统一规范化，避免系统不允许字符

### D. UI 入口

- 在歌曲详情弹窗加入单一操作：
  - 保存到本地
- 在设置页加入下载目录配置：
  - 默认系统下载目录
  - 自定义目录开关
  - 目录选择与恢复默认
- 第一版先做单曲下载，后续再扩展到多选批量下载

## 文件命名策略

- 默认格式：`艺术家 - 标题.扩展名`
- 若艺术家为空，则降级为 `标题.扩展名`
- 对 Windows / Android 不友好的字符进行替换
- 优先沿用原始 URI 或歌曲格式字段推断扩展名

## 风险与注意点

- Android 不同版本对公共目录写入权限策略不同
- 自定义目录选择依赖平台选择器能力，需要验证 Android 真机流程
- 远程文件可能没有稳定扩展名，需要做合理兜底
- 若已存在同名文件，需要避免直接覆盖用户已有文件

## 第一阶段交付范围

1. 单曲保存到 `Download/NagoMusic`
2. 在设置中配置单曲默认下载目录
3. 优先复用现有缓存
4. 歌曲详情页增加入口与提示

## 验证建议

1. 远程歌曲未缓存时，点击“下载到本地”，确认文件落到 `Download/NagoMusic`
2. 远程歌曲已缓存时再次下载，确认可以直接导出
3. 本地歌曲选择“保存到其他位置”，确认可以复制成功
4. 重名歌曲重复下载时，确认不会粗暴覆盖原文件
