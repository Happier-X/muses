# 专项优化：SongsPage 封面预取协调器拆分

## 1. 背景

在完成 `SongsPage` 可见列表控制层首轮拆分后，页面中仍然保留了大量与封面预取相关的逻辑，包括：

- 页面级封面缓存
- 进行中的加载任务表
- 队列与并发控制
- 预取范围调度
- 本地封面缓存写回逻辑

这些逻辑与页面展示强相关，但本质上属于“资源协调层”，不适合长期继续留在页面主体中。

## 2. 本次目标

本次拆分聚焦以下目标：

- 把 `SongsPage` 的封面预取和队列逻辑抽到独立协调器
- 页面只负责给出可见范围和触发时机
- 封面缓存、并发、预取窗口等逻辑在单独文件内维护

## 3. 拆分范围

本次纳入协调器的职责：

- 单曲封面加载
- 预取范围调度
- 并发队列
- 页面级封面缓存
- 本地缓存封面写回逻辑

暂不纳入的职责：

- 通用封面读取和压缩（已在 `ArtworkService`）
- 通用展示组件（已在 `ArtworkWidget`）

## 4. 预期收益

- `SongsPage` 页面文件继续变薄
- 封面相关性能问题有独立入口可调
- 页面逻辑和资源协调逻辑进一步分离
- 后续如果专辑页/艺术家页需要更复杂预取策略，也更容易复用思路

## 5. 验收标准

- 新增 `SongsArtworkCoordinator`
- `SongsPage` 不再持有主要的封面队列与预取实现
- 现有封面行为不变
- `flutter analyze` 通过
- 基础测试通过
