# 专项优化：统一页面初始化策略

## 1. 背景

在当前项目中，多个页面都存在各自独立的初始化流程，常见形式包括：

- `initState -> _init -> _load`
- `initState` 里直接 `_load()`
- `addPostFrameCallback + delayed + _init()`

这些实现虽然都能工作，但在体验和维护上已经出现明显问题：

- 不同页面的进入节奏不一致
- 有些页面会在转场动画期间执行重任务
- 相同的“首帧后初始化”逻辑在多个页面重复出现
- 后续继续优化时，容易再次出现某个页面回退到旧写法

## 2. 当前问题

结合近期性能排查，初始化逻辑分散主要带来以下影响：

- 切换到歌曲、专辑、艺术家等页面时，动画可能被初始化重任务抢占
- 返回首页或重新进入页面时，页面状态和加载时机不统一
- 页面文件中既有业务逻辑，也有大量重复的初始化样板代码
- 当需要调整初始化延迟、首帧策略或错误处理时，需要逐页修改

## 3. 本次优化目标

本次专项优化不做大范围架构改造，只解决“初始化入口不统一”的问题。

目标如下：

- 提供一套统一的页面延迟初始化机制
- 页面只声明“初始化要执行什么”，不重复写调度代码
- 默认把初始化放到首帧后，减少转场动画掉帧
- 为后续页面逐步迁移提供统一模板

## 4. 设计方案

本次计划新增一个轻量 mixin，用于 `StatefulWidget` 页面：

- 统一在 `initState` 中安排首帧后初始化
- 支持设置初始化延迟
- 自动处理 `mounted` 检查
- 页面只需要实现一个 `runDeferredInit()`

推荐使用方式：

```dart
class _ExamplePageState extends State<ExamplePage>
    with DeferredPageInitMixin {
  @override
  Duration get deferredInitDelay => const Duration(milliseconds: 120);

  @override
  Future<void> runDeferredInit() async {
    await _init();
  }
}
```

## 5. 首批落地范围

本次优先迁移以下页面：

- `SongsPage`
- `AlbumsPage`
- `ArtistsPage`
- `PlaylistsPage`
- `SourcePage`

选择这些页面的原因：

- 都是切页频率较高的主页面
- 都存在明显初始化逻辑
- 都是近期体验问题最集中的页面

## 6. 预期收益

- 页面切入时的行为更一致
- 转场动画期间的主线程压力更可控
- 初始化代码明显减少，页面更干净
- 后续继续做页面级性能优化时，有统一入口可扩展

## 7. 后续可继续扩展的方向

等这套机制稳定后，可以在这个基础上继续扩展：

- 初始化错误态统一处理
- 初始化中 loading 占位统一规范
- 页面恢复与缓存命中策略接入统一入口
- 页面埋点或性能统计统一接入

## 8. 本次落地标准

本次完成后，应满足：

- 主页面不再手写重复的 `addPostFrameCallback + delayed` 模板
- 至少首批几个高频页面迁移到统一初始化模式
- `flutter analyze` 通过
- 现有功能行为保持不变
