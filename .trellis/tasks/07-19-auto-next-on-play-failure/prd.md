# 播放失败后自动播放下一首

## 目标

当当前歌曲因文件不可访问、认证、网络或播放器错误而无法开始播放时，自动跳过失败歌曲并尝试队列中的下一首，避免播放流程停在错误状态。

## 已确认事实

- GitHub issue #40 要求“增加播放失败继续自动播放下一首的功能”。
- `playSong` 当前在 `AudioPlayerNative.play(...)` 抛错后设置安全错误文案、清理缓冲与媒体会话，但不会推进队列。
- 自然播放结束已通过 `advanceToNext()` + `playSong(nextSong)` 自动续播。
- 队列支持列表循环、单曲循环和随机顺序；`playSong` 使用 generation 防止快速切歌的旧请求回写。
- 播放失败错误可能包含认证信息，现有安全错误过滤必须保留。

## 需求

- `AudioPlayerNative.play` 确认失败且该请求仍是当前 generation 时，自动尝试队列中的下一首。
- 自动跳过必须沿用当前队列顺序，包括随机播放生成的顺序。
- 单次自动恢复过程中，每首歌曲最多尝试一次，避免列表循环、单曲循环或全部歌曲损坏时无限递归。
- 下一首播放成功后恢复正常 playing 状态，并清除上一首错误提示。
- 所有候选歌曲均失败或不存在下一首时停止自动推进，保留用户安全错误信息，不泄露原生错误、URL、认证头或密码。
- 用户快速手动切歌产生的过期播放失败不得触发自动推进或覆盖新歌曲状态。
- 保留切歌时的缓冲、歌词/封面 token、媒体会话和 WebDAV 凭据安全清理语义。

## 验收标准

- [x] 当前歌曲播放失败且队列有可播放下一首时，自动跳过并成功播放下一首。
- [x] 连续多首失败时继续尝试后续歌曲，每首最多一次。
- [x] 全队列均失败时终止恢复，不无限循环，并展示安全错误状态。
- [x] 随机模式按当前 shuffleOrder 推进。
- [x] 单曲循环模式不会反复重试同一首失败歌曲。
- [x] 过期 generation 的失败不自动切歌、不串改当前歌曲。
- [x] lint、build、unit test 与 git diff --check 全部通过，并有失败恢复回归测试。

## 范围外

- 不增加网络重试、指数退避或播放失败弹窗。
- 不修改音频解码器、WebDAV 下载方式或队列持久化格式。
- 不处理依赖升级。

## 已确认的产品决策

当处于单曲循环时，当前歌曲播放失败不能继续重试自身。失败恢复链临时忽略单曲循环，按当前 active order 尝试下一首；成功播放后保留用户的单曲循环配置，后续自然结束继续按单曲循环处理。
