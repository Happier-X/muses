# 沉浸式播放页面

## Goal

为当前音乐播放能力增加更完整的播放体验：底部迷你播放条在页面间常驻，点击后进入沉浸式播放页面；沉浸式播放页提供封面/歌曲信息、播放控制、进度控制，并通过 AMLL 实现背景与歌词展示，支持左滑切换到歌词页面。

## Confirmed Facts

- 当前播放器状态由 `src/features/player/controller.ts` 中的 feature-local `reactive<PlayerState>` 管理，没有 Pinia/Vuex。
- 当前 `PlayerState` 只包含 `status`、`currentSong`、`errorMessage`，尚无播放进度、时长、歌词、封面状态。
- 当前 `PlayerSongSnapshot` 只包含 `id`、`sourceId`、`sourceType`、`title`、`artist`、`album`，尚无封面、歌词路径或时长。
- 当前 `SongItem` 只包含标题、歌手、专辑、时长等基础元数据，尚无内嵌歌词、同目录歌词 URI/URL、封面字段或标签扫描状态。
- 当前本地/WebDAV 标签读取只返回标题、歌手、专辑、时长，尚未读取封面或歌词标签；Android `MediaMetadataRetriever` 可作为读取内嵌封面的原生边界。
- 当前 `MiniPlayer.vue` 已固定在底部并跨页面显示，但点击播放条本体尚未导航到播放页。
- 当前路由只有 `/tabs/*` 页面，尚无沉浸式播放页路由。
- 当前 `AudioPlayerNative` 只有 `play`、`pause`、`resume`、`stop`、`getState` 和 `stateChange`，尚无播放进度广播或 `seek` 方法。
- `package.json` 当前没有 AMLL 相关依赖。
- AMLL 官方 npm 包可用：`@applemusic-like-lyrics/vue`、`@applemusic-like-lyrics/core`、`@applemusic-like-lyrics/lyric`。
- WebDAV 密码等敏感信息必须只在 SecureStorage/native 边界使用，不得进入播放器状态、歌词状态、路由参数、日志或本地库元数据。

## Requirements

- 底部播放条应在主要页面常驻显示，已有正在播放/暂停/错误状态时不因切换页面消失。
- 点击底部播放条主体应进入沉浸式播放页面；点击播放/暂停/停止按钮仍执行原控制动作，不能误触导航。
- 沉浸式播放页面应包含两个横向可切换区域：
  - 信息/控制页：展示歌曲封面、标题、歌手/专辑、播放/暂停、停止、播放进度和拖动进度控制。
  - 歌词页：使用 AMLL 展示歌词，并使用 AMLL 背景能力，背景渲染器使用 `meshGradientRenderer`。
- 沉浸式播放页面应复用现有播放器状态与原生播放服务，不引入 Pinia/Vuex。
- 原生播放服务应提供当前播放位置、总时长和 `seek` 能力；前端应以适当频率同步进度并驱动进度条/歌词时间轴。
- Android 通知栏播放通知应展示播放进度，并在播放/暂停/seek 后保持与原生播放器进度一致。
- 歌词来源应支持歌曲标签内置 LRC 歌词，以及音频文件同目录同名 `.lrc` 歌词文件。
- 如果 WebDAV 入库时未选择扫描歌曲标签信息，应支持在播放歌曲时延迟扫描该歌曲标签并保存结果；每播放一首补扫一首，避免一次性补扫整个库。
- 延迟补扫应更新本地歌曲元数据和播放器当前展示信息，包括标题、歌手、专辑、时长、LRC 歌词和真实内嵌封面；不得阻塞播放启动，补扫失败时播放仍继续，并显示封面/歌词缺失降级状态。
- WebDAV 播放时应将音频文件缓存到应用私有缓存目录，后续播放同一首歌优先使用缓存文件实现快速启动；缓存必须有最大容量限制并按最近使用策略清理旧文件。
- 如果当前没有播放歌曲，沉浸式播放页面应展示空状态并提供返回入口，而不是崩溃。
- 如果歌曲没有歌词，应展示清晰的无歌词状态；AMLL 作为歌词渲染能力直接使用，不额外提供 AMLL 加载失败降级体验。
- 播放页、歌词/封面功能和 WebDAV 音频缓存不得泄露 WebDAV 密码、Basic Auth header 或 SecureStorage 值，也不得把封面大图 base64 或 WebDAV 音频缓存文件内容直接写入 `muses:songs`。
- 真实封面应由原生层提取后写入应用私有缓存文件，前端歌曲元数据只保存 `coverUri` 引用和必要更新时间；缓存缺失时允许播放/补扫流程重新提取。

## Acceptance Criteria

- [ ] 从任意主要 tab 页面点击底部播放条主体会进入沉浸式播放页面。
- [ ] 在底部播放条点击播放/暂停/停止按钮只控制播放，不触发进入播放页。
- [ ] 沉浸式播放页面的信息/控制页展示当前歌曲标题、歌手/专辑、真实内嵌封面；没有封面时展示占位封面，并提供播放/暂停、停止控制。
- [ ] 沉浸式播放页面展示进度条与当前/总时长；拖动进度条会 seek 到目标位置。
- [ ] Android 通知栏播放通知展示当前播放进度，播放/暂停/停止状态变化后进度显示保持合理同步。
- [ ] 左滑可从信息/控制页切换到歌词页；右滑可切回信息/控制页。
- [ ] 歌词页使用 AMLL 组件或 AMLL 渲染能力展示歌词，并使用 AMLL 背景的 `meshGradientRenderer`。
- [ ] 歌词优先使用歌曲标签内置 LRC；没有内置 LRC 时尝试使用同目录同名 `.lrc` 文件；都没有时展示无歌词状态。
- [ ] WebDAV 入库未扫描标签的歌曲在播放时会触发单曲延迟标签扫描，并把读取到的标题、歌手、专辑、时长、LRC 歌词和封面引用保存到本地库。
- [ ] WebDAV 歌曲播放时会缓存音频文件；再次播放同一首歌优先使用缓存文件，缓存总量超过上限时会清理旧缓存。
- [ ] 延迟补扫失败不会中断播放，也不会把 WebDAV 密码、Basic Auth 或音频缓存内容写入本地库、路由、日志或 UI 状态。
- [ ] 无当前歌曲、无歌词时都有可理解的 UI 状态。
- [ ] 单元测试覆盖迷你播放条导航、播放页基础渲染、播放控制、seek 参数、无歌词/无当前歌曲降级。
- [ ] `npm run test:unit -- --run`、`npm run lint`、`npm run build` 通过。

## Out of Scope

- 不实现播放队列、上一首/下一首。
- 不实现在线歌词搜索或跨来源自动匹配歌词，除非后续明确要求。
- 不新增 Pinia/Vuex。
- 不改变 WebDAV 密码存储策略。
