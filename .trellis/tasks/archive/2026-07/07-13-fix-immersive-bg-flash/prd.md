# 切歌沉浸式背景不闪默认

## Goal

Issue #20：沉浸式播放页切歌时，背景不要短暂回到默认渐变再切到下一首。

## Issue

- GitHub #20

## Root Cause

- AMLL `BackgroundRender` 挂在 `v-if="hasLyrics"`：切歌无词时整块卸载。
- 新歌 `coverUri` 为空时 `album` 为空，背景失去专辑图。

## Decisions

| 决策 | 结论 |
|------|------|
| 背景与歌词解耦 | 有当前曲即可渲染背景（有封面源时） |
| 粘性封面 | 切到无封面时短时保留上一首可用封面，新封面到位后更新；停播清空 |

## Requirements

1. **R1** 背景不因「暂无歌词」卸载。
2. **R2** 切歌无封面时不立刻清空背景用图（粘性上一张）。
3. **R3** 控制页封面图可与背景共用粘性逻辑（一致体验）。
4. **R4** 无当前曲时恢复默认/无粘性封面。
5. **R5** 单测覆盖：有词→无词切歌背景仍在；粘性封面。

## Out of Scope

- 跨曲封面渐变动画库
- 修改 AMLL 包源码

## Task Type

Lightweight（偏 UI 修复，design 可简）
