# 继续修复音源列表 issues

## 目标

继续处理当前仓库中与音源列表相关的未关闭 GitHub issues，恢复音源管理和重复扫描行为的完整性。

## 已确认事实

当前未关闭的相关 issues：

- #31：音源页面竖屏时列表重叠，扫描按钮被遮盖。
- #32：再次扫描空音源后，歌曲列表仍残留之前的歌曲。
- #33：音源列表缺少删除功能。
- #34：音源列表缺少编辑功能。

这些问题分别涉及响应式布局、扫描结果同步、音源删除和音源编辑，属于可独立验收的交付项，适合拆分为父任务下的子任务逐项处理。

## 需求

- 每个 issue 独立规划、实现、验证和关闭，避免把多种行为修改混入同一实现单元。
- 保持现有本地音源与 WebDAV 音源兼容。
- 涉及 WebDAV 凭据时，密码不得写入 `localStorage`、普通页面状态或日志。
- 不回归现有扫描设置约定：WebDAV 默认不读取标签，本地音源默认读取标签。

## 验收标准

- [x] #31：竖屏窄视口下音源卡片不重叠，扫描及后续操作按钮可见、可点击。（已完成：`a1e3c7a`，issue 已关闭）
- [x] #32：某音源重新扫描后，以本次扫描结果同步该音源的歌曲；当结果为空时，该音源旧歌曲不再残留，其他音源歌曲不受影响。
- [x] #33：用户可删除音源；删除后音源列表持久化更新，并按既有安全约定清理 WebDAV 安全存储凭据。
- [x] #34：用户可编辑音源的可编辑信息并持久化；WebDAV 凭据更新仍通过安全存储处理。（已完成：`158d1e4`，issue 已关闭）
- [x] 相关 lint、类型检查和测试通过。
- [x] 每个完成的 issue 均有独立可追踪的任务与提交，并已关闭对应 GitHub issue。

## 范围外

- 新增音源类型。
- 重构整个音乐库或播放器。
- 与上述四个 issue 无关的界面改版。

## 处理顺序（已确认）

按 `#31 → #32 → #33 → #34` 连续处理；每项独立规划、实现、验证、提交并关闭对应 GitHub issue。
