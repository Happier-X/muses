# 修复歌词活跃行居中

## 目标

修复沉浸式播放页歌词滚动定位，使当前播放歌词行稳定位于歌词可视区域中央，且不破坏歌词翻译、副行显示和手势交互。

## 需求

- AMLL 歌词当前行应定位在歌词可视区域的中心位置，而不是固定偏上位置。
- 保留歌词顶部歌曲信息、底部快捷操作和安全区布局。
- 不改变歌词解析、翻译合并、歌词点击跳转和左右面板切换行为。
- 横屏、竖屏和平板布局均应保持可用。

## 验收标准

- [x] `LyricPlayer` 的活跃行定位参数使当前行位于歌词区域视觉中心。
- [x] 当前行变化时自动滚动定位仍正常。
- [x] 翻译/音译副行仍与当前主行一起显示，不被底部操作遮挡。
- [x] 既有 lint、build、unit test 全部通过。

## 范围外

- 不重写 AMLL 滚动引擎。
- 不修改歌词解析和在线翻译匹配逻辑。
- 不处理其他 GitHub issue。
