# 修复播放模式与循环模式图标

## 目标

修复沉浸式播放页底部播放模式与循环模式按钮的图标展示，使图标能够准确反映当前模式，并让点击切换结果在视觉上立即可见。

## 已确认事实

- GitHub issue #38 标题指出：底部播放模式切换和循环模式切换图标展示有问题，图标看起来一直不变，并疑问点击是否能切换。
- `src/views/PlayerPage.vue` 当前循环模式已根据 `queueState.repeatMode` 在 `repeatOutline`（列表循环）与 `repeat`（单曲循环）间切换。
- 当前播放模式的 `shuffleIcon` 无论 `queueState.shuffleEnabled` 是否为真都固定返回 `shuffle`；但按钮 aria-label 会在“顺序播放”和“随机播放”间切换。
- `queue.ts` 已有并持久化 `repeatMode` 与 `shuffleEnabled` 的状态切换逻辑，本任务不改变队列语义。

## 需求

- 循环模式按钮的图标和 aria-label 必须与当前列表循环/单曲循环状态一致。
- 播放模式按钮的图标和 aria-label 必须与当前顺序播放/随机播放状态一致；当前状态切换后图标应立即更新。
- 点击按钮仍调用现有队列状态切换逻辑，不改变播放队列、shuffleOrder、repeatMode 或持久化协议。
- 保留图标按钮的可访问性标签和激活态样式。

## 验收标准

- [x] 列表循环与单曲循环使用可区分且稳定的图标，点击后图标和 aria-label 同步更新。
- [x] 顺序播放与随机播放使用可区分且稳定的图标，点击后图标和 aria-label 同步更新。
- [x] 既有队列状态、持久化和播放行为测试继续通过，并增加图标状态回归测试。
- [x] lint、build、unit test 与 git diff --check 全部通过。

## 范围外

- 不修改队列推进、随机排序、循环语义或播放器控制器。
- 不处理播放失败自动下一首和依赖升级。

## 待确认的产品决策

播放模式的“顺序播放”建议使用 `listOutline`（列表/顺序图标），随机播放使用 `shuffle`；循环模式继续使用现有 `repeatOutline` / `repeat`。这样两个状态在视觉上明确可区分，且不需要引入新依赖图标。
