# 专辑页改为卡片布局

## 目标

将专辑页从横向列表行改为以封面为主要视觉信息的卡片网格，提升专辑浏览时的辨识度和扫描效率。

## 已确认事实

- 当前 `src/views/AlbumsPage.vue` 使用 `ion-list` + `ion-item` 渲染专辑，每行左侧封面、右侧专辑信息。
- 专辑数据由 `groupSongsByAlbum` 聚合，现有卡片所需信息包括专辑名、歌曲数、艺术家摘要和专辑歌曲集合。
- `MCover` 支持传入数字尺寸，可复用现有封面加载、占位和圆角规则。
- 当前项目没有专辑详情页面或专辑点击路由，专辑页属于纯展示页面。
- 现有单元测试覆盖专辑聚合信息和空状态，但尚未约束布局结构。

## 要求

- 专辑页使用卡片网格替代 `ion-list` / `ion-item` 列表。
- 每张卡片展示正方形专辑封面、专辑名称、歌曲数量和艺术家摘要。
- 窄屏固定 2 列；宽屏在 `--muses-content-max-width` 限位内按可用宽度自动增加列数。
- 专辑名和艺术家摘要过长时不得撑破卡片或覆盖相邻内容。
- 保留现有专辑聚合、封面 URI 处理、生命周期刷新和空状态行为。
- 卡片布局同时适配窄屏和宽屏，并继续使用项目现有颜色、间距、圆角和内容宽度 token。
- 不改变艺术家页、歌曲页和导航结构。

## 验收标准

- [ ] 有专辑数据时以多列卡片网格展示，不再渲染 `ion-list` / `ion-item`。
- [ ] 窄屏固定 2 列，宽屏在内容宽度上限内自动增列。
- [ ] 每张卡片完整展示封面、专辑名、歌曲数量和艺术家摘要。
- [ ] 窄屏和宽屏下卡片尺寸稳定，文本不会溢出或相互遮挡。
- [ ] 无封面时继续展示 `MCover` 占位内容。
- [ ] 无专辑时继续展示“还没有专辑”空状态。
- [ ] 不新增专辑详情路由或虚假的可点击交互。
- [ ] 单元测试覆盖卡片结构以及现有聚合与空态行为。
- [ ] `npm run lint`、`npm run build` 和相关单元测试通过。

## 范围外

- 不新增专辑详情页、路由或点击播放行为。
- 不修改专辑聚合和封面解析业务逻辑。
- 不同步改造艺术家页或其他列表页面。
