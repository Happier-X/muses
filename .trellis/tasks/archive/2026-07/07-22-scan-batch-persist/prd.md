# 扫描批量持久化

## Goal

消除曲库扫描过程中逐首全量序列化、同步写入和广播造成的主线程阻塞。

## Requirements

- 扫描期间在内存中累积歌曲变更。
- 发现、标签读取、摘要计数和对账语义保持不变。
- 扫描成功完成后只对最终曲库做一次持久化和一次歌曲更新广播。
- 扫描失败不得提交未完成的半批数据。
- 进度通知节流，但阶段切换、失败和完成状态必须立即送达。
- `upsertSong` 的既有独立调用默认持久化行为保持兼容。

## Acceptance Criteria

- [x] 多首成功扫描对 `muses:songs` 至多一次写入、一次更新广播
- [x] 无变化扫描不产生无意义写入
- [x] 中途失败不提交半批数据
- [x] 摘要、WebDAV 降级和空扫描删除语义不回归
- [x] 单测、lint、build 通过
