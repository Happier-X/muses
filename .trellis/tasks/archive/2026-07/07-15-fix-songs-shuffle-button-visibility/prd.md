# 修复歌曲页随机播放按钮不可见

## 目标

让歌曲页的“随机播放全部”按钮在有歌曲时无需滚动到列表末尾即可看到和点击，同时保持按钮处于歌曲内容区与底部导航区域之间。

## 背景

- `src/views/SongsPage.vue:62` 当前把按钮放在歌曲列表后的普通文档流 `.bottom-actions` 中。
- 歌曲较多时，按钮位于整个列表末尾，进入页面时不可见；这不符合用户对“列表与 navbar 之间”的常驻操作入口预期。
- 移动端底部同时存在固定的 MiniPlayer 和 TabBar；宽屏端存在固定 MiniPlayer，因此按钮和列表内容都必须避让这些区域。
- 随机播放业务逻辑 `onShuffleAll` 已存在且无需改变。

## 需求

- 将“随机播放全部”调整为歌曲页底部常驻操作区，无需滚动到列表末尾即可看到。
- 移动端按钮位于 MiniPlayer 上方，不得被 MiniPlayer 或 TabBar 遮挡。
- 宽屏端按钮位于底部 MiniPlayer 上方，并遵守现有内容最大宽度与居中规则。
- 为歌曲列表预留足够底部空间，确保最后一首歌曲不会被常驻按钮、MiniPlayer 或 TabBar 遮挡。
- 保持现有横向整行描边样式（`fill="outline"` + `expand="block"`）、按钮文案、图标、可访问性标签、无歌曲禁用态和 `onShuffleAll` 行为不变。
- 不调整全局 MiniPlayer、TabBar 或随机播放队列逻辑。

## 验收标准

- [ ] 有多首歌曲且列表可滚动时，刚进入歌曲页即可看到“随机播放全部”按钮。
- [ ] 滚动歌曲列表时，按钮保持在歌曲内容区底部，并可正常点击。
- [ ] 移动端按钮不被 MiniPlayer 或 TabBar 遮挡；宽屏端不被 MiniPlayer 遮挡。
- [ ] 列表滚动到底后，最后一首歌曲完整可见且不被底部常驻元素遮挡。
- [ ] 无歌曲时按钮仍显示为禁用态。
- [ ] 点击按钮后的随机播放全部行为保持不变。
- [ ] 相关单元测试、类型检查与构建通过。

## 范围外

- 修改随机队列算法或播放器行为。
- 改造全局底部导航与 MiniPlayer 布局。
- 调整搜索、歌曲菜单或跳转当前播放按钮。
