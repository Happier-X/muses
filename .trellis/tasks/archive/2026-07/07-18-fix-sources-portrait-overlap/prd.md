# 修复音源页竖屏列表重叠

## 目标

修复音源页面在竖屏窄视口下虚拟列表行高估算不足，导致音源卡片互相覆盖并遮挡扫描按钮的问题。

## 需求

- 音源卡片的虚拟列表位置必须基于实际渲染高度，而不是固定高度估算。
- 竖屏窄视口下，长路径或换行内容不能造成下一张卡片覆盖当前卡片。
- 扫描按钮在每张卡片中保持可见、可点击。
- 保持现有虚拟列表能力，不改变音源数据和扫描业务逻辑。

## 验收标准

- [x] 音源页面在窄屏/竖屏下连续展示多个音源时，卡片之间不重叠。
- [x] 长内容导致卡片高度变化时，后续卡片位置仍正确。
- [x] 每张卡片的扫描按钮可见且可点击。
- [x] 桌面/平板布局和空列表状态不回归。
- [x] `npm run lint`、`npm run build`、`npm run test:unit` 通过。

## 范围外

- 不新增编辑、删除入口。
- 不修改扫描器和歌曲存储逻辑。
