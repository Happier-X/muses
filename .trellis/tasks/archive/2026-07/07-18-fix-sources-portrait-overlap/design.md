# 技术设计

## 根因

`src/views/SourcesPage.vue` 使用 `@tanstack/vue-virtual` 渲染绝对定位卡片，但虚拟器固定使用 `estimateSize: () => 148`。音源卡片实际高度会因窄屏宽度、路径内容和 Ionic 内边距变化而超过估算值，因此后续行的 `translateY` 过小，发生重叠。

## 方案

保留虚拟列表结构，为每个实际渲染的卡片提供虚拟器的测量元素引用和索引属性，使 TanStack Virtual 在挂载/尺寸变化后使用真实高度重新计算后续行位置；同时保留合理的估算值作为首次布局占位。卡片宽度使用现有左右约束，避免引入与本 issue 无关的布局改造。

## 验证与回滚

通过 lint、类型检查、单元测试和人工窄屏检查验证。若测量引用导致 Ionic 卡片更新异常，可回滚到单独的固定最小行高方案，但优先采用实际测量以覆盖动态高度。
