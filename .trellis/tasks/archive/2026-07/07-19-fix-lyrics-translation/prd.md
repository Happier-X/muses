# 修复歌词翻译

## 目标

修复沉浸式播放页歌词翻译相关问题：翻译不显示、翻译与原歌词错位，以及当前播放行变化时翻译副行不高亮。

## 已确认现象

用户确认 GitHub #36 同时包含以下现象：

1. 翻译有时不显示。
2. 翻译与原歌词时间轴错位。
3. 当前播放行变化时，翻译副行没有同步高亮。

## 需求

- 对在线翻译和同歌词内双语行，可靠地把译文挂到对应原歌词行的 `translatedLyric` 字段。
- 支持常见 LRC 时间戳格式，包括厘秒、百分秒、毫秒和多时间戳行；译文与原文存在小幅时间偏差时仍能正确匹配。
- 不应把相邻但属于两句独立原歌词的同时间行错误合并为翻译。
- 翻译显示开关打开时，AMLL 能渲染翻译副行；关闭时只隐藏翻译/音译，不破坏主歌词和逐字时间轴。
- 当前主歌词行激活时，其翻译副行应同步获得明确的激活样式；非激活行保持较低视觉权重。
- TTML、LRC、YRC、QRC 歌词解析和在线歌词回退链继续兼容。
- 不把翻译密码、请求凭据或不安全数据写入持久化存储或日志。

## 验收标准

- [x] 有译文的歌曲在翻译开关打开时能显示译文；无译文时不伪造内容。
- [x] 译文与对应原歌词行匹配，允许合理的时间戳偏差，并覆盖多时间戳和毫秒格式。
- [x] 翻译开关关闭后主歌词仍正常显示，再次打开后译文恢复。
- [x] 播放推进到下一行时，原歌词和翻译副行同步高亮；非激活翻译不会保持激活样式。
- [x] TTML 内置翻译和平台 `tlyric` 翻译均可正常显示。
- [x] 既有歌词解析、歌词点击跳转、活跃行居中和左右滑动行为不回归。
- [x] `npm run lint`、`npm run build`、`npm run test:unit -- --run` 通过，并补充匹配与播放页回归测试。

## 范围外

- 不更换在线歌词供应商。
- 不重写 AMLL 歌词滚动引擎。
- 不处理进度条、播放模式或自动下一首问题。
