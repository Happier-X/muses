# 实施计划

1. 在 `src/theme/variables.css` 增加所有普通 navbar 标题的统一绝对居中规则，排除 `size="large"`。
2. 检查并清理各页面仅用于普通标题居中的重复局部样式，确保大标题不被误居中。
3. 增加单元测试或结构测试，覆盖：全局规则存在、普通标题命中、`size="large"` 被排除、长标题安全空间和按钮交互层不被破坏。
4. 更新前端组件规范，记录 Material Design 标题 flex 偏移陷阱和统一实现约定。
5. 运行 `npm run test:unit -- --run`、`npm run lint`、`npm run build`、`git diff --check`。
6. 独立检查所有页面、详情页、播放队列和 Modal 标题，以及折叠大标题零回归。

## 风险与回滚点

- 风险：绝对标题覆盖按钮或截断空间不足。检查最长动态歌单标题及文字“关闭”按钮；按钮保持高层级。
- 风险：选择器误命中折叠大标题。必须使用 `:not([size="large"])` 并有测试保护。
- 回滚：移除全局标题规则并恢复原页面局部样式。
