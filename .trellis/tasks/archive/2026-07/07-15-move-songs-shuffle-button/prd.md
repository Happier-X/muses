# 调整歌曲页随机播放按钮位置

## 目标

将歌曲页“随机播放全部”入口从底部常驻区域移到顶部 Navbar 正下方，并靠左固定展示，使歌曲列表滚动时入口仍保持可见，页面信息层级和操作位置接近椒盐音乐。

## 背景

- `src/views/SongsPage.vue:62` 当前通过 `ion-content` 的 `slot="fixed"` 将 `.bottom-actions` 常驻在 MiniPlayer 上方。
- 当前按钮为 `expand="block"` 的整行描边按钮，与用户期望的顶部左侧入口不一致。
- 已确认采用推荐样式：紧凑、无整行描边的“shuffle 图标 + 随机播放全部文字”按钮。
- 用户所说的“顶部 navbar”指歌曲页顶部标题栏；目标顺序应为：顶部 Navbar → 固定的随机播放入口 → 可滚动歌曲列表。
- 用户进一步确认“固定在 Navbar 下面”：随机播放入口不能作为会随歌曲列表滚走的普通内容。
- 现有随机播放业务逻辑、无歌曲禁用态及可访问性语义已经具备。

## 需求

- 将“随机播放全部”从底部固定操作区移除，放到顶部 Navbar 正下方、歌曲列表上方的固定操作区。
- 固定操作区应与顶部 Navbar 一起保持在页面顶部；歌曲列表独立滚动时，入口始终可见。
- 随机播放入口在现有内容限宽区域内左对齐，移动端与宽屏端均保持一致的相对位置。
- 入口采用紧凑、无整行描边的按钮样式，由 shuffle 图标与“随机播放全部”文字组成，不横向撑满容器。
- 保持现有 `aria-label="随机播放全部"`、shuffle 图标、文案、无歌曲禁用态和 `onShuffleAll` 行为不变。
- 删除仅用于底部常驻操作区的定位样式，并恢复列表底部仅避让 MiniPlayer、TabBar 等实际固定元素所需的空间。
- 不修改随机队列算法、播放器逻辑、Navbar、MiniPlayer、TabBar、搜索按钮及歌曲行交互。

## 验收标准

- [ ] 歌曲页视觉顺序为：顶部 Navbar → 左对齐的固定随机播放入口 → 可滚动歌曲列表。
- [ ] 滚动歌曲列表时，随机播放入口固定在 Navbar 正下方，不随列表滚走。
- [ ] 随机播放入口不再固定在页面底部，也不再显示为横向撑满的底部操作条。
- [ ] 移动端和宽屏端均在 Navbar 下方左侧显示该入口，并遵守现有内容最大宽度规则。
- [ ] 无歌曲时入口仍可见且处于禁用态。
- [ ] 点击入口后，现有“清空队列 → 加入全部歌曲 → 开启随机 → 播放乱序首曲”行为保持不变。
- [ ] 最后一首歌曲滚动到底后不被 MiniPlayer 或 TabBar 遮挡。
- [ ] 相关单元测试、类型检查与构建通过。

## 范围外

- 修改随机播放算法或播放队列行为。
- 仿制椒盐音乐的其他页面或交互。
- 调整全局 Navbar、MiniPlayer 或 TabBar 布局。
