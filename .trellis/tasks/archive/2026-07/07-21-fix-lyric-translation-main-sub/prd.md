# 修复歌词翻译主副行与高亮

## Goal

修复沉浸式歌词页翻译相关回归，使：

1. **主行始终是原文**（如英文/日文），**副行是译文**（如中文），而不是默认中文主行 + 英文副行；
2. **关闭翻译**后只保留原文主行，不残留「只剩中文」；
3. **当前活跃行高亮可持续**（播放进度内保持），不再「闪一下就灭」。

## Background / 用户报告

- 默认：中文成了主行，英文成了副行（主副颠倒）。
- 默认：当前播放行高亮一下即消失。
- 点翻译开关后：只展示中文，高亮同样只闪一下。

## 根因假设（实现中验证）

1. **主副颠倒**：`mergeDuplicateTimestampTranslations` 对同时间戳、不同文字体系的双主行，固定「前一行主、后一行译」。当 LRC 顺序为「中文在前、原文在后」时，中文被当成主行。
2. **关翻译只剩中文**：关翻译仅清空 `translatedLyric`，主行若已是中文则只剩中文——由 (1) 引起。
3. **高亮闪一下**：合并后可能未正确保留配对行的 `endTime` / 词级时间，导致 AMLL 活跃窗口极短；或主副颠倒后与播放进度不匹配。实现时需核对合并后的 `startTime`/`endTime`/`words`，必要时取配对行 `endTime` 的较大值。

## Requirements

### R1. 双行合并主译判定
- 同时间戳双语合并时，必须按**语义**选主行，不能只靠文件顺序。
- 约定（外语歌 + 中文译的常见场景）：
  - 若一对中一行是 **Han（中文）**、另一行是 **非 Han**（Latin / 假名 / Hangul 等）→ **非 Han 为主行**，Han 为 `translatedLyric`。
  - 若一对中一行是 **kana（日文假名）**、另一行是 **Han** → 已由「非 Han 优先」覆盖时，假名侧为主；纯汉字日文与中文难区分时，可保持顺序优先并加单测标注边界。
  - 其它可区分脚本对：仍要求 `isLikelyTranslationPair`；主行默认前一行，除非命中「Han 应为译」规则。
- 已有独立 `lyricsTranslation`（tlyric）挂载的行：不得被双行合并再次颠倒主译。

### R2. 翻译开关
- 开启：主行原文 + 副行译文（`translatedLyric`）。
- 关闭：仅主行原文，`translatedLyric` / `romanLyric` 清空（沿用 `applyLyricTranslationVisibility`）。
- 关闭后不得出现「只剩译文、无原文」。

### R3. 活跃行高亮
- 播放至某行时间窗内，该行主行（及开启翻译时的副行）保持 AMLL 激活样式，直至进入下一行或离开窗口。
- 合并双语行时：结果行的 `endTime` 不得短于配对两行中较合理的结束时间（建议 `Math.max(current.endTime, next.endTime)`，词级 `words` 以主行为准）。
- 不修改 `node_modules` / AMLL 源码；样式仍用 `.FmKaba_*` 实际类名。

### R4. 测试
- 补充/更新 `lyrics-merge-translation` 单测：
  - **中文在前、英文在后** 同时间戳 → 主行 Hello、译 你好。
  - **英文在前、中文在后** → 仍正确（回归）。
  - 日文假名/混排 + 中文译 → 原文主、中文译。
  - 关翻译后主行仍为原文。
  - 合并后 `endTime` 取 max（若实现采用）。
- 现有同语言不合并、tlyric 挂载、容差用例保持通过。

### Out of Scope
- 不重做在线歌词匹配源优先级。
- 不改 AMLL 依赖版本。
- 不发版（除非另行要求）。
- 不解决「纯汉字歌词无法区分原文/译文」的无脚本差异场景（可文档说明限制）。
