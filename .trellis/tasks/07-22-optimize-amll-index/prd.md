# AMLL 索引匹配降载

## Goal

降低首次下载解析约 1.44 MiB JSONL 索引和后续逐曲全索引评分造成的主线程尖峰。

## Requirements

- 保持远程索引、串行源优先级、评分阈值和匹配结果兼容。
- 对索引构建可复用的规范化标题候选结构，匹配先缩小候选再精确评分。
- 大文本解析分片让出事件循环；若 Worker 会显著增加构建/测试复杂度，优先分片实现。
- 并发仍只下载和构建一次；失败允许重试。
- 不把索引打包进 APK，不修改第三方库。

## Acceptance Criteria

- [x] 常见 exact/contains 查询不再扫描全部 3125 条记录
- [x] 分片解析期间可让出事件循环
- [x] 既有评分边界、网络失败、并发缓存测试通过
- [x] lint、unit、build 通过
