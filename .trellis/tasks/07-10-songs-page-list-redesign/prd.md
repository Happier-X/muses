# 改造歌曲页列表样式

## 背景

用户希望将当前歌曲页改造成更接近目标截图的列表样式。截图无法在当前模型环境中直接解析，但用户补充了核心要求：顶部右侧是搜索入口；每一行展示封面、歌名、艺术家 - 专辑、更多按钮。

## 目标

- 歌曲页顶部右侧提供搜索入口。
- 歌曲列表每行展示：
  - 封面
  - 歌名
  - `艺术家 - 专辑`
  - 更多按钮
- 保持现有歌曲播放能力不回归。
- 保持手机端和平板端基础可用，避免破坏当前主导航布局。

## 范围

包含：

- 修改 `src/views/SongsPage.vue` 的页面头部和列表行布局。
- 使用已有歌曲数据字段展示歌名、艺术家、专辑信息。
- 如已有封面字段则展示封面；否则提供稳定占位封面。
- 增加更多按钮 UI，至少保持可点击/可扩展，不要求本次实现完整菜单功能。
- 去掉歌曲列表行底部分隔线/下划线。
- 歌曲播放触发元数据补充后，列表能刷新并展示最新元数据。
- 构建验证。

不包含：

- 新增全局搜索功能完整实现，除非现有代码已有可复用能力。
- 新增歌曲更多菜单的完整业务操作。
- 重构播放器或音源扫描逻辑。
- 发布新版本。

## 验收标准

- [x] 歌曲页顶部右侧有搜索入口。
- [x] 歌曲行显示封面、歌名、`艺术家 - 专辑`、竖向三点更多按钮。
- [x] 歌曲行不显示底部分隔线/下划线。
- [x] 点击歌曲仍可播放。
- [x] 获取到元数据后列表能刷新展示新封面/歌名/艺术家/专辑。
- [x] 空列表/加载/错误状态仍可正常展示。
- [x] `npm run build` 通过。
