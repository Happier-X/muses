# 添加音源扫描入库

## 目标

在音源列表中为每个音源提供“扫描”入口。用户点击某个音源的扫描按钮后，可以选择是否读取音乐标签，并在扫描过程中通过弹窗看到进度；扫描完成后，符合条件的歌曲被写入本地歌曲库，供后续歌曲页使用。

## 背景与已确认事实

- 当前音源页位于 `src/views/SourcesPage.vue`，已支持添加本地文件夹和 WebDAV 文件夹。
- 音源元数据类型位于 `src/features/sources/types.ts`，目前只有 `local` 和 `webdav` 两类音源。
- 音源元数据通过 `src/features/sources/storage.ts` 保存到 `localStorage` 的 `muses:sources`，WebDAV 密码保存到 SecureStorage，不进入 `localStorage`。
- WebDAV 目录浏览通过 `src/features/sources/webdav.ts` 和 Android 原生 `WebDav` Capacitor 插件执行 `PROPFIND`，当前只返回目录，不返回文件。
- 当前仓库没有歌曲库、歌曲存储模型、递归扫描器、音频标签读取能力或歌曲页数据接入实现。
- 当前依赖中没有音频元数据解析库；本任务要求真实读取音乐标签，因此需要新增前端依赖或原生能力。

## 需求

- R1：音源列表每一项卡片上显示一个“扫描”按钮，按钮与现有虚拟列表布局兼容。
- R2：点击“扫描”后先弹出扫描设置弹窗，至少允许选择“是否读取音乐标签”。
- R3：确认扫描后显示进度弹窗，扫描期间能看到当前阶段、已发现/已入库数量，以及完成或失败状态。
- R4：扫描应把该音源下支持的音频文件写入本地歌曲库，歌曲记录至少包含唯一标识、来源音源 ID、来源类型、路径/URL、展示标题、创建/更新时间。
- R5：扫描 WebDAV 音源时必须使用已保存的 `credentialKey` 读取密码，不能把密码写入 `localStorage` 或日志。
- R6：扫描支持重复执行；同一音源同一路径的歌曲应更新或跳过，不能产生无限重复记录。
- R7：读取音乐标签为可选项；关闭时使用文件名作为标题，开启时必须真实解析可访问音频文件的标题/歌手/专辑等标签，单个文件解析失败时回退文件名并计入失败/降级摘要。
- R8：扫描失败时要向用户展示明确错误，并保留已安全入库的数据；不能导致现有音源丢失。

## 验收标准

- [ ] 每个音源卡片都有可点击的“扫描”按钮。
- [ ] 点击扫描会先出现包含“读取音乐标签”开关的设置弹窗。
- [ ] 扫描进行中会出现进度弹窗，展示阶段、计数和当前处理项或摘要。
- [ ] 扫描完成后会展示成功摘要，包含入库/更新/跳过/失败数量。
- [ ] 歌曲库数据持久化后刷新页面仍可读取。
- [ ] WebDAV 密码仍只通过 SecureStorage 获取，不进入歌曲库或音源 metadata。
- [ ] 重复扫描同一音源不会重复插入同一路径歌曲。
- [ ] 单元测试覆盖歌曲存储去重、音频文件过滤、WebDAV 文件解析、标签读取成功/失败摘要或扫描摘要逻辑。

## 暂定技术边界

- 优先沿用 `src/features/<feature>/` 组织方式，新增歌曲库和扫描能力的 feature helper，不引入全局状态库。
- 本任务聚焦“从音源扫描到本地歌曲库持久化”和音源页扫描 UX；歌曲页完整展示/播放可作为后续任务，除非用户明确要求纳入本任务。
- 本任务包含真实音乐标签读取；允许对单个无法解析的文件回退文件名，但不能把标签读取整体做成占位实现。

## 已决策问题

- Q1：本任务必须实现真实音乐标签读取。
- Q2：本任务必须同时支持本地文件夹和 WebDAV 音源扫描入库，并在开启读取音乐标签时真实读取两类音源的标签。
