# 添加音源扫描入库实施计划

## 实施步骤

1. 调研并选定真实音乐标签读取方案，优先验证浏览器端解析库是否能处理 WebDAV 下载数据和本地可访问文件。
2. 新增本地文件夹递归扫描与元数据读取的 Android 原生能力，覆盖 `content://` 树 URI 下的音频文件枚举和标签读取。
3. 新增歌曲库类型与持久化模块，支持校验、加载、保存、按 `sourceId + path` 去重 upsert。
4. 扩展 WebDAV 模块，保留现有目录浏览 API，同时新增能返回目录和文件条目的列表能力，并提供标签读取所需的安全下载/读取入口。
5. 新增标签读取模块，真实解析标题、歌手、专辑、时长等字段，单文件失败时回退并计数。
6. 新增扫描器模块，按音源类型递归遍历音频文件、过滤扩展名、读取标签、生成歌曲记录、回调进度并返回摘要。
7. 在音源卡片新增“扫描”按钮，接入设置弹窗和进度弹窗。
8. 补充单元测试：歌曲库校验与去重、音频文件过滤、WebDAV 文件解析、标签读取降级、扫描摘要。
9. 运行 `npm run test:unit`、`npm run build`，必要时运行 `npm run lint`。

## 验证命令

- `npm run test:unit`
- `npm run build`
- `npm run lint`
- `cd android && ./gradlew :app:compileDebugKotlin`

## Android 手工验证清单

- 本地目录：Android 10/11/13+ 至少一台设备或模拟器，选择含多层子目录、中文路径、空格路径、大量文件的音乐目录。
- 本地标签：验证 MP3/M4A/FLAC 等常见格式的标题、歌手、专辑、时长读取；验证标签异常文件会回退文件名并计入降级。
- WebDAV 扫描：验证中文路径、空格路径、嵌套目录、401/403 认证失败、大文件、标签读取失败降级。
- 安全检查：扫描完成后检查 `localStorage.muses:songs` 不含 WebDAV 密码、Basic Auth Header 或 SecureStorage 值。
- 重复扫描：同一音源重复扫描不重复插入，同一路径标签变化时更新并保留 `createdAt`。

## 风险点

- 本地目录递归扫描能力当前未在仓库出现；本任务需新增原生能力，工作量和 Android 兼容性风险较高。
- 真实音乐标签读取当前没有依赖；引入解析库会增加体积、异步下载、跨平台兼容风险，并可能需要限制文件大小或分片读取。
- WebDAV 递归扫描大目录可能耗时较长，需要进度反馈和错误隔离。
- 虚拟列表卡片高度可能因按钮增加而变化，需要同步调整 `estimateSize` 和样式。

## 启动前检查

- 用户已确认本任务必须真实读取音乐标签。
- 用户已确认本地文件夹与 WebDAV 都必须支持扫描入库和真实标签读取。
- 用户确认本任务是否包含歌曲页展示，默认不包含。
- `implement.jsonl` 和 `check.jsonl` 已加入前端规范上下文。
