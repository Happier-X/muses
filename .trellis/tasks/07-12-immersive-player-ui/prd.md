# 沉浸式播放页 UI 复刻

## Goal

将现有沉浸式播放页（`PlayerPage.vue`）的视觉与布局，复刻用户提供的参考截图风格，让全屏播放体验更接近现代音乐 App 的沉浸感，同时保留现有播放能力与交互约定。

## Background

- 当前 `src/views/PlayerPage.vue` 已具备：AMLL 动态背景、封面、歌名/歌手、进度条 seek、上一曲/播放暂停/下一曲、循环/随机/队列入口、左右滑歌词面板、下滑关闭 overlay。
- 参考图为手机竖屏截图（约 1200×2670），深色背景 + 封面 + 中部信息 + 底部控制区。
- 播放逻辑、overlay 状态、媒体会话、队列持久化不在本任务范围内。

## Requirements

### 必须

1. 按参考截图复刻沉浸式播放页**控制页**的窄屏竖屏布局与视觉风格（间距、层级、封面比例/圆角、标题排版、进度条形态、主控制区与次要控制区排布）。
2. 保留现有能力：
   - 封面展示（拒绝 `data:` / `blob:` / base64）
   - 进度 seek
   - 上一曲 / 播放暂停 / 下一曲
   - 循环模式切换、随机模式切换、队列入口
   - AMLL 歌词背景（有歌词时）与 fallback 背景
   - 左右滑切换「控制页 / 歌词页」
   - 下滑关闭播放器 overlay
3. 空状态（无当前歌曲）仍可用，文案清晰；不再依赖顶部「返回」入口。
4. 宽屏（`min-width: 768px`）继续保持封面控制 + 歌词双栏并排，不回归到仅单栏。
5. 不破坏 `App.vue` 异步加载 `PlayerPage`、overlay 非路由、Android 返回键优先级等既有约定。
6. 更新/补充必要的单元测试，使 `npm run test:unit`、`npm run lint`、`npm run build` 通过。

### 已确认

- 复刻范围：**A. 视觉重排**——只重排/重绘现有控件，不新增音量/收藏/分享等功能控件。
- 顶部区域：**C. 无顶部文案，仅安全区留白**——不保留「正在播放」标题，也不新增显式顶部关闭控件；关闭依赖现有下滑手势与系统返回。
- 次要控制：**A. 纯图标横排**——循环 / 随机 / 队列仅保留图标，模式状态用高亮或透明度表达；`aria-label` 保留无障碍语义。
- 歌词页：**A. 本轮只改控制页**——歌词页只做必要适配，不单独做视觉复刻。

### 非目标

- 不改原生播放/通知/媒体会话逻辑。
- 不改迷你播放条业务逻辑（除非为配合视觉必须微调）。
- 不新增加密源、歌词搜索、在线封面等新功能。
- 不主动提交/推送 git（除非用户明确要求）。
- 不对本轮歌词页做独立视觉重设计。

## Target Layout（控制页）

窄屏自上而下：

1. 顶部安全区留白（无标题、无返回按钮）
2. 大封面（居中、圆角、阴影）
3. 歌名（主标题）+ 歌手/专辑（副标题）+ 可选元数据状态提示
4. 进度条 + 左右时间
5. 主控制：上一曲 / 播放暂停（强调） / 下一曲
6. 次要控制：循环 / 随机 / 队列（纯图标）
