# 修复进度条拖动与点击

## 目标

修复沉浸式播放页进度条交互：拖动时已播放填充区域必须同步移动，点击轨道任意位置应跳转到对应播放位置。

## 已确认现象

GitHub #37：拖动进度条时只有圆点移动，填充区域没有移动，进度条无法点击跳转。

## 需求

- 进度条圆点和已播放填充区域始终使用同一个播放位置状态。
- 用户点击轨道任意位置时，应按点击位置计算目标秒数并调用统一的 `seekPlayback`。
- 拖动过程中应提供即时视觉反馈；提交 seek 时应保留缓冲区限制和 seek 手势保护。
- 缓冲已知时不得跳转到未缓冲区域；缓冲未知时按歌曲总时长限制。
- 不回归歌词点击 seek、当前进度同步、自然结束判定和上一曲/下一曲手势隔离。

## 验收标准

- [x] 拖动圆点时，已播放填充区域同步移动。
- [x] 点击进度轨道任意位置会跳转到对应位置，而不是只有拖动圆点有效。
- [x] 目标位置按总时长和已缓冲终点正确限制。
- [x] 进度条交互不会误触发上一曲、下一曲或面板切换。
- [x] 既有 lint、build、unit test 通过，并有进度条组件行为回归测试。

## 范围外

- 不修改播放器队列、自然结束或 WebDAV 缓冲实现。
- 不处理播放模式图标、播放失败自动下一首和依赖升级。
