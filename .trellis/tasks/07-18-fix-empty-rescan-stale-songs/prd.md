# 空扫描后同步清理旧歌曲

## 目标

修复音源重新扫描后，若本次发现结果为空或文件减少，歌曲库仍残留该音源旧歌曲的问题。

## 需求

- 某音源扫描成功后，以本次发现的文件路径集合同步该音源歌曲：新增/更新仍走既有 upsert，不在集合中的旧歌曲应被移除。
- 本次发现结果为空时，该音源下旧歌曲应全部清除。
- 仅影响当前扫描音源的歌曲；其他音源歌曲不得被改动。
- 发现阶段失败（无法列出文件）时，不得因“未发现文件”误删已有歌曲。
- 保持本地与 WebDAV 扫描路径一致；不改变 `readTags` 默认策略与凭据安全约定。
- 扫描摘要应能反映移除数量，便于用户确认同步结果。

## 验收标准

- [x] 音源首次有歌曲后，再次扫描发现 0 个音频文件，该音源歌曲从库中清除。
- [x] 再次扫描发现文件减少时，仅移除该音源下未再出现的路径对应歌曲。
- [x] 其他音源歌曲在同步过程中保持不变。
- [x] 列表/发现失败时不清理旧歌曲。
- [x] `npm run lint`、`npm run build`、`npm run test:unit` 通过，并有覆盖空扫描与部分删除的回归测试。

## 范围外

- 音源删除/编辑 UI（#33/#34）。
- 播放队列中对已删除歌曲的自动清理策略（除非现有逻辑已自动失效）。
- 扫描进度 UI 的大幅改版。
