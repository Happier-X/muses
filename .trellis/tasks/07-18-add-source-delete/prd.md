# 音源列表支持删除

## 目标

为音源列表补充删除能力，让用户可以从音源页移除不再需要的本地或 WebDAV 音源，并清理该音源关联的持久化数据。

## 需求

- 音源卡片必须提供明确的删除入口，且不影响现有扫描入口。
- 删除前必须二次确认，避免误删。
- 确认删除后，音源列表持久化更新，刷新后该音源不再出现。
- 删除 WebDAV 音源时，必须移除对应 `credentialKey` 的 SecureStorage 密码；密码不得进入 `localStorage`、普通页面状态或日志。
- 删除音源时，应同步清理该音源下的歌曲记录，避免歌曲列表残留孤儿歌曲；其他音源歌曲不受影响。
- 删除失败时应给出用户可见错误，不应误报成功。

## 验收标准

- [x] 每个音源卡片都有可点击的删除入口，扫描按钮仍可用。
- [x] 点击删除后出现确认弹窗；取消时不改动音源、歌曲或 WebDAV 凭据。
- [x] 确认删除本地音源后，`muses:sources` 删除该音源，`muses:songs` 删除该 `sourceId` 下歌曲。
- [x] 确认删除 WebDAV 音源后，同时调用 SecureStorage 删除该音源 `credentialKey`，且不存储/暴露密码。
- [x] 删除某音源不会影响其他音源及其歌曲。
- [x] `npm run lint`、`npm run build`、`npm run test:unit` 通过，并有存储层回归测试覆盖本地/WebDAV 删除。

## 范围外

- 音源编辑功能（#34）。
- 批量删除音源。
- 清理播放队列或当前播放状态中的孤儿歌曲（除非现有逻辑已自动过滤）。
