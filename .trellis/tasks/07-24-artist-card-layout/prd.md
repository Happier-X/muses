# 艺术家页改为卡片布局

## 目标

将艺术家页从列表行改为卡片网格，卡片上方展示艺术家头像，下方展示艺术家名称和统计信息，提升艺术家浏览时的辨识度并与专辑页卡片布局保持一致。

## 已确认事实

- 当前 `src/views/ArtistsPage.vue` 使用 `ion-list` + `ion-item` 渲染艺术家列表。
- 当前宽屏艺术家页通过 `.list-grid` 包裹 `ion-list` 并用 `display: contents` 实现多列列表。
- 艺术家数据由 `groupSongsByArtist` 聚合，`ArtistSummary` 已包含 `name`、`songCount`、`albumCount` 和 `songs`。
- `src/views/AlbumsPage.vue` 已采用卡片网格布局，可作为本次视觉与响应式行为参考。
- `MCover` 可复用封面加载、占位与尺寸控制；艺术家没有独立头像字段，可从该艺术家的歌曲封面中取第一张可展示封面作为头像来源，无封面时显示占位。
- 当前项目没有艺术家详情页或艺术家点击路由，艺术家页属于纯展示页面。

## 要求

- 艺术家页使用卡片网格替代 `ion-list` / `ion-item` 列表。
- 每张艺术家卡片上方展示头像区域，下方展示艺术家名称、歌曲数量和专辑数量。
- 头像优先使用艺术家歌曲中的第一张可展示封面；无有效封面时使用 `MCover` 占位。
- 艺术家头像应呈现为头像语义的圆形视觉，而不是专辑封面的方形视觉。
- 窄屏固定 2 列；宽屏在 `--muses-content-max-width` 限位内按可用宽度自动增加列数。
- 艺术家名称和统计文本过长时不得撑破卡片或覆盖相邻内容。
- 保留现有艺术家聚合、生命周期刷新和空状态行为。
- 卡片布局继续使用项目现有颜色、间距和内容宽度 token。

## 验收标准

- [ ] 有艺术家数据时以多列卡片网格展示，不再渲染 `ion-list` / `ion-item`。
- [ ] 卡片上方展示圆形艺术家头像区域，下方展示艺术家名称、歌曲数量和专辑数量。
- [ ] 窄屏固定 2 列，宽屏在内容宽度上限内自动增列。
- [ ] 艺术家无可用封面时继续展示稳定占位内容。
- [ ] 窄屏和宽屏下卡片尺寸稳定，文本不会溢出或相互遮挡。
- [ ] 无艺术家时继续展示“还没有艺术家”空状态。
- [ ] 不新增艺术家详情路由或虚假的可点击交互。
- [ ] 单元测试覆盖卡片结构以及现有聚合与空态行为。
- [ ] `npm run lint`、`npm run build` 和相关单元测试通过。

## 范围外

- 不新增艺术家详情页、路由、点击播放或筛选行为。
- 不修改艺术家聚合业务逻辑。
- 不修改专辑页、歌曲页和导航结构。
