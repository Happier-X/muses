# 音源列表支持编辑

## 目标

为音源列表补充编辑能力，让用户无需删除并重新添加音源，即可修改本地或 WebDAV 音源的可编辑信息，并安全持久化修改结果。

## 已确认事实

- GitHub issue #34 标题为“音源列表缺少编辑功能”，issue 正文未补充更细范围。
- 音源分为本地与 WebDAV 两类，元数据保存在 `localStorage` 的 `muses:sources`。
- WebDAV 密码仅通过 SecureStorage 保存，普通音源元数据只保留 `credentialKey`。
- 本地音源当前包含名称与目录路径；WebDAV 音源包含名称、服务地址、用户名、目录路径和凭据引用。
- 编辑功能不得改变音源 `id`、`type` 和 `createdAt`，以维持歌曲与音源的关联身份。

## 需求

- 每个音源卡片提供明确的编辑入口，且不影响现有扫描、删除入口。
- 编辑表单按音源类型展示可编辑字段，并预填当前非敏感信息。
- 本地音源允许修改显示名称和目录；重新选择目录时保留原音源 `id`，以维持歌曲关联身份。
- WebDAV 音源允许修改显示名称、服务地址、用户名和目录；密码输入框不得回填现有密码。
- WebDAV 密码留空时保留原凭据；输入新密码时只写 SecureStorage，不得写入 `localStorage`、普通页面状态之外的持久化数据或日志。
- WebDAV 服务地址、用户名、目录或密码变化时，保存前必须使用“新输入密码或原安全凭据”在线验证目标目录；仅修改显示名称时无需联网。
- 在线验证失败时不得更新音源元数据或安全凭据。
- 保存失败时给出用户可见错误，且不得误报成功。
- 编辑一个音源不得影响其他音源。
- 本地目录或 WebDAV 服务地址、用户名、目录发生变化时，不立即清理已有歌曲；保留到用户下次成功扫描，再按扫描对账规则清理旧记录。
- 编辑后刷新页面仍显示最新信息。

## 验收标准

- [x] 本地与 WebDAV 音源卡片均有可点击的编辑入口，扫描和删除按钮仍可用。
- [x] 编辑表单预填当前非敏感字段，不读取或展示已有 WebDAV 密码。
- [x] 保存后保留原音源 `id`、`type`、`createdAt`，仅更新允许编辑的字段。
- [x] WebDAV 密码留空时不调用 SecureStorage 写入；提供新密码时更新原 `credentialKey` 对应值。
- [x] WebDAV 连接字段变化时先在线验证目标目录，失败不修改元数据或凭据；仅改名不要求联网。
- [x] 密码不出现在 `muses:sources`、日志或用户可见错误详情中。
- [x] 保存失败时列表保持可恢复状态并显示错误，不误报成功。
- [x] 编辑某音源不改变其他音源，也不立即清理该音源已有歌曲；下次成功扫描再对账。
- [x] `npm run lint`、`npm run build`、`npm run test:unit -- --run` 通过，并有存储层测试覆盖本地/WebDAV 编辑与凭据行为。

## 范围外

- 修改音源类型。
- 批量编辑音源。
- 新增音源类型。
- 与编辑无关的音源页改版。
