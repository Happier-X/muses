# 首页最近内容功能更新计划

## 背景

当前首页仅提供音乐库统计和分类入口，缺少“最近听过 / 最近查看”的快速回到能力。为了缩短用户从首页进入常用内容的路径，首页底部新增以下三个分区：

1. 最近歌曲
2. 最近歌单
3. 最近专辑

## 本期目标

在首页卡片区下方新增三个列表分区，支持用户快速进入最近使用的内容。

## 第一版口径

### 最近歌曲

- 数据来源：播放统计表 `song_stats`
- 排序方式：按 `lastPlayedMs` 倒序
- 展示数量：最多 6 首
- 交互：点击后直接从当前最近歌曲列表开始播放，并定位到所点歌曲

### 最近歌单

- 第一版数据来源：歌单表 `playlists`
- 排序方式：按 `createdAtMs` 倒序
- 展示数量：最多 6 个
- 交互：点击进入歌单详情页
- 说明：当前仓库里没有“最近打开歌单”的独立埋点，因此本期先按“最近创建/变更顺序可感知的歌单”落地；后续如果增加访问记录表，再升级为真实“最近打开歌单”

### 最近专辑

- 数据来源：最近播放歌曲结果
- 聚合方式：从最近播放歌曲中按专辑名去重聚合
- 排序方式：按该专辑最近一次被播放的歌曲时间倒序
- 展示数量：最多 6 个
- 交互：点击进入专辑详情页

## 交互与视觉

- 分区放在首页入口卡片下方
- 每个分区包含标题、空状态和纵向列表
- 保持现有首页卡片风格，不引入新的复杂视觉层级
- 优先保证移动端单列阅读体验

## 实现步骤

1. 补充播放统计服务中的最近歌曲查询方法
2. 在首页加载逻辑中并行拉取最近歌曲、最近歌单和最近专辑数据
3. 新增首页最近内容分区组件
4. 接入歌曲播放、歌单详情跳转、专辑详情跳转
5. 跑格式化与静态分析

## 后续可迭代项

1. 增加“最近打开歌单 / 最近打开专辑”访问埋点，替换第一版近似口径
2. 为首页最近内容增加缓存，减少重复计算
3. 支持横向卡片样式和“查看更多”入口
4. 支持按照当前音源过滤最近歌曲与最近专辑
